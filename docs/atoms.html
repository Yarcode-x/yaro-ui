Átomos

Los elementos más básicos del sistema. No dependen de otros componentes de @yaro/ui y se usan dentro de moléculas y organismos. Todos son standalone, usan ChangeDetectionStrategy.OnPush y :host { display: contents }.

YaroButton yaro-button

Botón de acción principal. Tres variantes visuales y tres tamaños. Soporte nativo para estado de carga con spinner animado.

Variantes
Tamaños — primary
Estados
// Variantes
<yaro-button variant="primary">Guardar</yaro-button>
<yaro-button variant="secondary">Cancelar</yaro-button>
<yaro-button variant="ghost">Eliminar</yaro-button>

// Tamaños
<yaro-button size="sm">Small</yaro-button>
<yaro-button size="lg">Large</yaro-button>

// Disabled
<yaro-button variant="primary" [disabled]="true">Disabled</yaro-button>

// Loading
<yaro-button variant="primary" [loading]="isSaving">Guardando…</yaro-button>

// Submit en formulario
<yaro-button type="submit" variant="primary">Enviar</yaro-button>

YaroInput yaro-input

Campo de texto con label, hint y manejo de errores integrado. Compatible con ngModel y ReactiveFormsModule vía ControlValueAccessor.

◈
ControlValueAccessor — usa directamente [(ngModel)] o [formControl]. No necesitas manejar eventos (input) manualmente.
Tal como aparece en el inventario
Ingresa un valor numérico válido 
// Básico con ngModel
<yaro-input
  label="Nombre"
  placeholder="Ej: Juan Pérez"
  [(ngModel)]="nombre"
/>

// Con hint
<yaro-input
  label="Email"
  type="email"
  hint="Nunca compartimos tu email"
  [(ngModel)]="email"
/>

// Con error (reactivo)
<yaro-input
  label="Precio"
  type="number"
  [error]="form.get('precio')?.hasError('min') ? 'El precio debe ser mayor a 0' : ''"
  [formControl]="form.get('precio')"
/>
API — @Inputs · ControlValueAccessor
PropiedadTipoDefaultDescripción
labelstring''Texto del label visible sobre el input
placeholderstring''Texto de placeholder del input nativo
typestring'text'Tipo HTML: text, email, number, password, etc.
errorstring''Mensaje de error. Si no está vacío, pinta el borde rojo
hintstring''Texto de ayuda debajo del input (se oculta si hay error)
inputIdstringautoID para el for del label. Se genera automáticamente si no se pasa
YaroToggle yaro-toggle

Interruptor binario on/off. Compatible con formularios vía ControlValueAccessor. Ideal para activar/desactivar configuraciones o estados como turnos activos.

◈
ControlValueAccessor — usa [(ngModel)] o [formControl]. También acepta [value] + (valueChange) si prefieres control manual.
Turno activo
Notificaciones
Modo oscuro
Deshabilitado 
// Con ngModel (recomendado)
<yaro-toggle label="Turno activo" [(ngModel)]="turnoActivo" />

// Con value + Output (control manual)
<yaro-toggle
  label="Notificaciones"
  [value]="notificaciones"
  (valueChange)="onToggle($event)"
/>

// En formulario reactivo
<yaro-toggle
  label="Activo"
  [formControl]="form.get('activo')"
/>
API — @Inputs / @Outputs · ControlValueAccessor
PropiedadTipoDefaultDescripción
@Inputlabelstring''Texto visible junto al toggle
@InputvaluebooleanfalseEstado del toggle (on/off)
@OutputvalueChangeEventEmitter<boolean>—Emite el nuevo valor al cambiar
YaroAvatar yaro-avatar

Representación visual de un usuario. Muestra una imagen si src es válido; si no, genera iniciales automáticamente desde name. Si la imagen falla al cargar, hace fallback a iniciales.

Tamaños con iniciales
YV
YV
YV
Variantes de color (por inicial)
AL
CR
MP
JS
Con imagen
// Solo con nombre → genera iniciales automáticamente
<yaro-avatar name="Ana López" />
<yaro-avatar name="Carlos Ruiz" size="lg" />

// Con imagen (fallback a iniciales si falla)
<yaro-avatar
  src="/api/users/42/photo"
  name="María Pérez"
  size="md"
/>

// En una lista de usuarios
@for (user of users; track user.id) {
  <yaro-avatar [src]="user.photo" [name]="user.name" size="sm" />
}
API — @Inputs
PropiedadTipoDefaultDescripción

namestring''Nombre completo. Se usan las dos primeras letras como iniciales
size'sm' | 'md' | 'lg''md'sm=28px · md=40px · lg=52px
YaroDelta yaro-delta

Indicador de variación numérica con flecha de dirección y color semántico. Verde = positivo, rojo = negativo. El flag inverted invierte los colores para métricas donde menos es mejor (como devoluciones o errores).

Normal — positivo es bueno
↑ +12.5% ↓ -3.2% — 0.0%
Invertido — menos es mejor (ej: devoluciones)
↓ -5 devoluciones ↑ +8 devoluciones
Con unidad personalizada
↑ +$45.000 ↑ +24 uds 
// Porcentaje (default)
<yaro-delta [value]="12.5" unit="%" />
<yaro-delta [value]="-3.2" unit="%" />

// Sin decimales
<yaro-delta [value]="24" unit=" uds" [decimals]="0" />

// Invertido — positivo es malo (devoluciones, errores)
<yaro-delta [value]="8" unit=" devoluciones" [inverted]="true" />

// Dinámico — desde signal
<yaro-delta [value]="ventasDelta()" unit="%" />
API — @Inputs
PropiedadTipoDefaultDescripción
valuenumber0Valor de la variación. Positivo = flecha arriba, negativo = abajo
unitstring''Sufijo del valor: %, uds, km, etc.
decimalsnumber1Número de decimales a mostrar
invertedbooleanfalseInvierte los colores: positivo = rojo, negativo = verde. Para métricas donde menos es mejor
