Instalación

@yaro/ui es una librería Angular standalone privada, alojada en GitHub Packages. Puede instalarse como paquete npm desde cualquier proyecto Angular, o consumirse directamente vía path alias dentro del monorepo Nx.

📦
@yaro/ui
github.com/ValentinaX63/yaro-ui
Privado · v1.0.0
Elige tu método
Dependiendo de dónde viva tu proyecto, hay dos formas de usar la librería.
Proyecto externo Repo separado · instala vía npm
Monorepo Nx Mismo workspace · path alias
ℹ
Requisitos: Angular 17+ · Node 18+ · Acceso al repositorio ValentinaX63/yaro-ui en GitHub · Personal Access Token con scope read:packages.
✓
Más simple. Si tu proyecto ya está en el mismo monorepo Nx que libs/yaro-ui, solo necesitas el path alias. No hay nada que instalar.
1 · Personal Access Token
Para instalar paquetes privados desde GitHub Packages necesitas un token con el scope read:packages. Genera uno en GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic).
⚠
Nunca comitees el token. Guárdalo como variable de entorno o en tu ~/.npmrc global. El .npmrc del proyecto solo debe referenciar la variable.
→
Guarda el token como variable de entorno
Agrégala a tu ~/.zshrc o ~/.bashrc para que persista.
~/.zshrc
export GITHUB_TOKEN=ghp_tuTokenAqui
2 · Archivo .npmrc
Crea un .npmrc en la raíz de tu proyecto para que npm sepa que los paquetes con scope @yaro vienen de GitHub Packages. Agrega .npmrc al .gitignore si contiene tokens directos, o usa la variable de entorno como se muestra aquí.
2
Crear .npmrc en la raíz del proyecto
El token se lee de la variable de entorno — nunca se hardcodea.
.npmrc (raíz del proyecto)
@yaro:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=${GITHUB_TOKEN}
3 · Instalar el paquete
3
npm install
Con el .npmrc configurado, la instalación es igual que cualquier paquete npm.
terminal
npm install @yaro/ui
4 · Configurar Angular
Agrega los estilos globales y las fuentes. Solo se hace una vez por proyecto.
4a
Estilos globales en angular.json
Agrega yaro-ui.scss antes que tus estilos de aplicación.
angular.json
"styles": [
  "node_modules/@yaro/ui/styles/yaro-ui.scss", // ← primero
  "src/styles.scss"
]
4b
Google Fonts en index.html
Outfit (principal) y JetBrains Mono (numérico/código).
src/index.html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link
  href="https://fonts.googleapis.com/css2?family=Outfit:wght@300;400;500;600;700&family=JetBrains+Mono:wght@400;500&display=swap"
  rel="stylesheet"
>
5 · Importar componentes
Todos los componentes son standalone. Impórtalos directamente en el array imports del componente que los use.
5
Importar desde @yaro/ui
Sin módulos intermedios. Solo importa lo que necesitas.
mi-feature.component.ts
import {
  YaroButtonComponent,
  YaroBadgeComponent,
  YaroInputComponent,
  YaroCardComponent,
} from '@yaro/ui';

@Component({
  selector: 'app-mi-feature',
  standalone: true,
  imports: [YaroButtonComponent, YaroBadgeComponent, YaroInputComponent, YaroCardComponent],
  template: `
    <yaro-card title="Ventas hoy">
      <yaro-button variant="primary">Guardar</yaro-button>
    </yaro-card>
  `,
})
export class MiFeatureComponent {}
Monorepo Nx — path alias
Si tu proyecto ya vive en el mismo workspace que libs/yaro-ui, solo necesitas registrar el path alias. Angular resuelve el import directamente desde el código fuente — no necesitas build previo ni npm install.
1
Path alias en tsconfig.base.json
tsconfig.base.json
{
  "compilerOptions": {
    "paths": {
      "@yaro/ui": ["libs/yaro-ui/src/index.ts"]
    }
  }
}
2
Estilos globales en angular.json
angular.json
"styles": [
  "libs/yaro-ui/src/styles/yaro-ui.scss",
  "src/styles.scss"
]
✓
Los pasos de Google Fonts e importar componentes son iguales en ambos métodos. Ver pasos 4b y 5 arriba.
Publicar nueva versión
Cuando agregues o modifiques componentes, sigue estos pasos para publicar una nueva versión en GitHub Packages y que los proyectos que la usan puedan actualizarse.
cmd
Build + publish en 3 comandos
Recuerda actualizar el número de versión en libs/yaro-ui/package.json antes de publicar.
terminal · desde la raíz del monorepo
# 1. Bump de versión (edita package.json manualmente o con npm version)
npm version patch --prefix libs/yaro-ui   # 1.0.0 → 1.0.1

# 2. Build de la librería con ng-packagr
nx build yaro-ui

# 3. Publicar en GitHub Packages
cd dist/libs/yaro-ui
npm publish
Flujo completo
1
Haz tus cambios en libs/yaro-ui/src/
2
Actualiza la versión en libs/yaro-ui/package.json — sigue semver: patch para fixes, minor para features, major para breaking changes
3
Corre nx build yaro-ui y verifica que no haya errores
4
Publica con npm publish desde dist/libs/yaro-ui
5
Haz commit y push al repo: git commit -m "chore: bump v1.0.1"
6
En los proyectos que la consumen: npm update @yaro/ui
